# 03.py - MongoDB 连接测试脚本

## 脚本功能

用于测试 MongoDB 连接配置，验证 JDBC 连接参数是否正确。特别针对 StarRocks 连接 MongoDB 时 `authSource` 是否需要配置进行测试。

## 主要功能

1. 测试两种连接方式：带 `authSource=admin` 和不带 `authSource`
2. 检查网络连接是否正常
3. 验证认证是否通过
4. 列出数据库中的集合
5. 如果存在 `demo_nodes` 集合，显示文档数量和样本数据

## 测试配置

测试的 MongoDB 地址：`10.34.106.203:37020`
- 数据库：`jarvis`
- 用户名：`jarvis_starRocks_wr`

## 代码结构

### `test_mongo_connection()` 函数

- 参数：测试名称、连接字符串
- 返回：`(成功/失败, 错误信息)`
- 自动处理错误分类：
  - 认证失败
  - 网络连接失败
  - 其他错误

## 执行结果分析

根据测试结果给出不同结论：

1. **带 authSource 通过，不带不通过**  
   💡 结论：用户存储在 admin 库，JDBC URI 必须包含 `authSource=admin`

2. **两种方式都失败**  
   ⚠️ 需要检查：
   - 用户名密码是否正确
   - 网络连通性（`telnet 10.34.106.203 37020`）
   - 用户权限配置

3. **两种方式都通过**  
   💡 结论：`authSource` 不是必须的

## 使用方法

